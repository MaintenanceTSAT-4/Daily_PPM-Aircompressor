# Daily PPM — Air Compressor (Web App)

ระบบบันทึกตรวจเช็คประจำวันเครื่องอัดอากาศ พร้อม Login, Approval Workflow,
E-Signature และการออกรายงาน PDF + Audit Log — สร้างด้วย **HTML/CSS/JS ล้วน + Supabase**

## คุณสมบัติ
- เข้าสู่ระบบด้วย **รหัสพนักงานอย่างเดียว** (ไม่ต้องใส่อีเมล/รหัสผ่าน) — Admin เพิ่มรหัสพนักงานให้ล่วงหน้า
- บันทึกตรวจเช็ครายวัน ตามหัวข้อมาตรฐาน 10 รายการ (Pressure Load/Unload, อุณหภูมิ, Oil Separator, กระแส Motor/Fan, ระดับน้ำมัน, Manual Drain, ทำความสะอาด, Running Hour) — เตือนเมื่อค่าหลุดช่วง
- Workflow: แบบร่าง → ส่งขออนุมัติ → อนุมัติ/ตีกลับ พร้อม **การแจ้งเตือนบน Dashboard** (กระดิ่ง)
- **E-Signature 2 แบบ**: วาดลายเซ็น (canvas) และลงชื่อแบบพิมพ์ + วันเวลา/ผู้ลงนาม
- โหมด **Admin**: เพิ่ม/ลบ/ปิดใช้ เครื่องจักรและหัวข้อตรวจเช็ค + กำหนดสิทธิ์ผู้ใช้
- ออก **PDF** ต่อรายการ ฝังลายเซ็นทั้งสองฝ่าย + แนบ **Audit Log** ครบถ้วน (รองรับฟอนต์ไทย)

## โครงสร้างไฟล์
```
index.html          หน้า Login (กรอกรหัสพนักงาน)
setup.html          ตั้งค่า Admin คนแรก (ใช้ครั้งเดียว แล้วลบทิ้ง)
app.html            หน้าแอปหลัก (Tab: บันทึก / Dashboard / อนุมัติ / Admin)
css/styles.css      สไตล์ทั้งหมด
js/config.js        ★ ใส่ค่า Supabase URL/Key ที่นี่
js/signature.js     วิดเจ็ตลายเซ็น (วาด/พิมพ์)
js/daily-check.js   ฟอร์มบันทึกตรวจเช็ค
js/dashboard.js     Dashboard + Approvals + หน้ารายละเอียด
js/admin.js         จัดการเครื่องจักร/หัวข้อ/ผู้ใช้
js/pdf.js           สร้าง PDF + ฝังลายเซ็น + Audit Log
js/app.js           ตัวควบคุม (session, เมนู, แจ้งเตือน)
supabase/schema.sql ★ สคริปต์สร้างฐานข้อมูล + RLS + ข้อมูลตั้งต้น
supabase/migration_employee_code.sql  สำหรับ DB เดิม: เพิ่มคอลัมน์รหัสพนักงาน
```

## วิธีติดตั้ง (3 ขั้นตอน)

### 1) ตั้งค่า Supabase
1. สร้างโปรเจกต์ที่ https://supabase.com
2. เปิด **SQL Editor** → วางเนื้อหาทั้งหมดของ `supabase/schema.sql` → กด **Run**
   (จะสร้างตาราง, RLS, และหัวข้อตรวจเช็คมาตรฐาน 10 รายการ + เครื่องตัวอย่าง 2 เครื่อง)
3. ไปที่ **Settings → API** คัดลอก *Project URL* และ *anon public key*

### 2) ใส่ค่าในแอป
แก้ไฟล์ `js/config.js`:
```js
const SUPABASE_URL = "https://xxxx.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGci...";
```

> **หากเคยรันสคีมาเดิมไปแล้ว** ให้รัน `supabase/migration_employee_code.sql` เพิ่ม เพื่อเพิ่มคอลัมน์รหัสพนักงาน

### 2.5) ปิด Confirm email (สำคัญมาก)
เพราะระบบใช้ "รหัสพนักงาน" แทนอีเมลจริง ต้องปิดการยืนยันอีเมล ไม่งั้นบัญชีจะใช้ไม่ได้:
Supabase → **Authentication → Providers → Email** → ปิดสวิตช์ **Confirm email** → Save

### 3) เปิดใช้งาน
เปิดผ่านเว็บเซิร์ฟเวอร์ (อย่าเปิดไฟล์ตรงๆ เพราะ Auth ต้องใช้ http):
```bash
# จากในโฟลเดอร์นี้
python -m http.server 5500
# แล้วเปิด http://localhost:5500/index.html
```
หรือใช้ส่วนขยาย **Live Server** ของ VS Code

## การเข้าสู่ระบบ (รหัสพนักงานอย่างเดียว)
- พนักงานพิมพ์ **รหัสพนักงาน** ที่หน้าแรกแล้วกดเข้าสู่ระบบ ไม่ต้องใช้รหัสผ่าน
- เบื้องหลังระบบจะผูกรหัสพนักงานกับ Supabase Auth โดยอัตโนมัติ (สร้างอีเมลภายใน `รหัส@ppm.local` + รหัสผ่านที่อนุมานจากรหัส) — ผู้ใช้ไม่เห็นส่วนนี้
- ⚠️ ข้อจำกัด: เนื่องจากไม่มีรหัสผ่าน ใครรู้รหัสพนักงานก็ล็อกอินแทนกันได้ เหมาะกับแท็บเล็ตหน้างาน หากต้องการความปลอดภัยสูงขึ้นให้เพิ่ม PIN ภายหลัง

## ตั้งค่าผู้ดูแลระบบ (Admin) คนแรก
1. เปิด **`setup.html`** (เช่น `http://localhost:5500/setup.html`) กรอกรหัสพนักงาน + ชื่อ แล้วกด "สร้างบัญชี Admin"
2. กลับไปที่ `index.html` เข้าสู่ระบบด้วยรหัสพนักงานนั้น จะเห็นเมนู **Admin**
3. **ลบไฟล์ `setup.html`** ทิ้งเพื่อความปลอดภัย
4. จากเมนู Admin → แท็บ **ผู้ใช้งาน** เพิ่มรหัสพนักงานคนอื่น (operator/approver/admin) ได้เลย

## บทบาทผู้ใช้
| สิทธิ์ | ทำอะไรได้ |
|--------|-----------|
| operator | บันทึกตรวจเช็ค, ลงลายเซ็นผู้ตรวจ, ส่งขออนุมัติ, ออก PDF |
| approver | ทุกอย่างของ operator + อนุมัติ/ตีกลับ + ลงลายเซ็นผู้อนุมัติ |
| admin | ทุกอย่าง + จัดการเครื่องจักร/หัวข้อ/สิทธิ์ผู้ใช้ |

## หมายเหตุทางเทคนิค
- PDF สร้างฝั่ง client ด้วย `html2canvas` + `jsPDF` (rasterize DOM จึงรองรับฟอนต์ไทย) — รวมลายเซ็นเป็นรูปและแนบ Audit Log
- ความปลอดภัยข้อมูลบังคับด้วย **Row Level Security** ของ Supabase (เขียน machines/check_items ได้เฉพาะ admin ฯลฯ)
- การแจ้งเตือนใช้การ poll ทุก 30 วินาที — หากต้องการ realtime สามารถต่อ Supabase Realtime ภายหลังได้
- ลายเซ็นเก็บเป็น base64 PNG ในตาราง `signatures`; ทุกการกระทำสำคัญถูกบันทึกใน `audit_logs`
